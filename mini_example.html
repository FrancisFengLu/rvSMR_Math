RARE & MR-CARV · 数据生成走查 (mini example)

n=4, p 缩小到 2-3, 让每个矩阵都肉眼可读 · 配套 methods_diagram.html · comparator_validation.html

本页用一个 n=4 的玩具版本, 一步一步把 RARE 和 MR-CARV 的 simulation pipeline 走一遍. 每个矩阵都写在屏幕上, 每个乘法都展开, 目的就是让"为什么这么生数据"看得见、摸得着.

为了直观, 我们用 seed=1 + n=4 的玩具版本; 真实仿真 n=50000 且基因型是 Gaussian copula 切的, 这里直接用整数 dosage. 数值都四舍五入到 1–3 位小数, 方便用脑子追.

诚实说出来的简化 —
  • 基因型 G: 真实是 Gaussian copula → 按 MAF 分位切; 这里我们 直接写下 整数 dosage {0,1,2}, 大致符合 MAF.
  • 混杂 U: 真实是 N(0, I) 然后按 h² 缩放; 这里只取 1 位小数的 N(0,1) 近似值.
  • 所有效应系数 (δ, κ, θ, ψ, γ) 都说成是 "从对应方差的 normal 里抽一次", 数值已经四舍五入到 2 位小数.
  • RARE 总样本量 n=4 算 Var/Cov 时用 n (population estimator), 不用 n-1.
  • 真因果 β₁ = 0.01 (RARE) 和 β = 0.04 (MR-CARV) 都是小的非零 alternative, 不是 null.
  • RARE 还有两个源码里的细节本玩具版没体现 (审计标出): (a) 真实仿真总样本是 100K, 拆成 50K summary panel + 50K calculation panel 两份独立抽; (b) Y 的噪声方差是 异方差 的, 按样本的 PRS 排序缩放 σᵢ = (PRSᵢ − PRSmin)/(PRSmax − PRSmin) + 0.05 — 这是 simulationl.R 里没在论文写出来的设计选择.

Section 1 · RARE 玩具版

我们要识别 $X_1 \to Y$ 的因果效应 $\beta_1$, 同时 RARE 还显式建立了 潜在混杂 U、correlated pleiotropy κ、uncorrelated pleiotropy θ 三层 "干扰". 所以下面会看到 3 个矩阵叠在一起决定 $X_1$ 和 $Y$.

Step 1

设定真因果参数 — 我们要识别的量

主曝光 → outcome 的真效应 (在 power cell 里设非零, 在 type-I cell 里设 0):

β₁ = 0.01   (主曝光 X₁ → Y, 这是我们要估的)
β₂ = 0.20   (次曝光 X₂ → Y, 固定大值, 用来制造 MVMR 设定)
Step 2

解释方差超参 h² (决定每条通路的方差预算)

RARE 通过 $h^2$ 控制每个矩阵的整体方差. 玩具版直接给数:

h²_γ₁ = 0.10  (common IV 对 X₁ 的总解释)
h²_δ = 0.08  (rare IV 对 X₁ 的总解释)
h²_κₓ = 0.02  (CHP: G → X₁ → Y 之外的 G → X₁ 路径)
h²_θ, h²_κᵧ, h²_ψ = 0.02  (其他通路, 都固定 0.02)
为什么要 h²?
RARE 的 prior 不是"每个 SNP 独立 N(0,1)", 而是"每个 G→X 通道总方差 = h²". 所以单个系数的方差 = h² / p_active. 这样不管 p 多大, 模拟的"信号强度"都保持可比.

Step 10

组装 X₁ — 矩阵形状视角

$$ \underbrace{X_1}_{n \times 1} \;=\; \underbrace{G\delta}_{(n \times p)(p \times 1)} \;+\; \underbrace{G\kappa_x}_{(n \times p)(p \times 1)} \;+\; \underbrace{U\psi_x}_{(n \times p_u)(p_u \times 1)} \;+\; \underbrace{\varepsilon_1}_{n \times 1} $$

三个 matrix-vector 乘积 + 一个 noise 向量, 全是 (n × 1) 列向量, 在样本维度 element-wise 加起来 → X₁.

放大单样本 i (= 1..4):
  (X₁)i = (Gδ)i + (Gκₓ)i + (Uψₓ)i + (ε₁)i
其中 (Gδ)i = Σj=1..5 Gij δj   (一个 dot product, 把 p=5 维 collapse 成标量)
输入数据 G, U 系数 δ, κₓ, ψₓ 噪声 ε 输出 X₁ · = matrix-vector 乘 (collapse p) + = element-wise 加 (shape 必须一致)
行/列向量惯例 (默认打开)
参数表 (δ, κₓ, ψₓ 等) 在源码里以长度 p 的 1-D 数组存; 在矩阵乘法 G·δ 里它们当 column vector 用, 即 right-multiply 一个 (n × p) 的 G, 得到一个长度 n 的 column vector. 上图所有 (p × 1) 的窄盒子都是这个意思.

为什么 X₂ 这么干净 (没 CHP 没 confounder)?
RARE 论文 Eq. 7–8 和 simulationl.R 都是这样设置: X₂ 这一臂没有 CHP 也没有 U 通道. 只有 X₁ 才同时受 IV 信号 (δ), CHP 上游 (κₓ), 和潜在混杂 (U ψₓ) 影响. 原因是 RARE 想造的最难场景就是 "想识别的因果效应 β₁ 对应的 X₁ 受混杂污染, 而控制变量 X₂ 是干净的 instrument-only 通道".

组装 Y — 矩阵形状视角 (6 个 n-vector 相加)

6 个 (n × 1) 列向量, element-wise 加. 注意 β₁, β₂ 是标量, 把 X₁, X₂ 缩放 (scalar × vector); 其它都是 matrix·vector.

6 个标量加起来. (Gκᵧ)i = Σj Gij κᵧ,j.
G, U κᵧ, θ, ψᵧ X₁, X₂, Y εᵧ · matrix-vector (collapse p / p_u) + element-wise (vector shape match)
为什么 Y 里要塞这么多项?
RARE 的核心卖点就是它 显式 建模 CHP (κᵧ) 和 UHP (θ) — 这两条 "直接 G → Y" 通道是 valid-instrument 假设的最常见破坏方式. 在仿真里 把它们生进去, 才能测算法在 pleiotropy 存在时还能不能识别 β₁. 混杂 U 通过 ψ_y 进 Y 制造 confounding bias. 对比一下: MR-CARV 仿真里 Y = β X + ε, 上面这些都没有.

Step 13

Summary stats — 矩阵形状视角 (对每列 G[:,j] 跑一次)

$$ \hat\beta_{x,j} \;=\; \frac{\mathrm{Cov}(G_{:,j},\, X_1)}{\mathrm{Var}(G_{:,j})}, \quad j = 1, \dots, p \;\;\Longrightarrow\;\; \hat\beta_x \in \mathbb{R}^{p} = \mathbb{R}^{5} $$

每个 SNP 跑一次单变量回归 ⇒ 把 G 的 p=5 列, 每列 collapse 成 1 个标量 β̂. 输出是一个 (p × 1) 列向量.

放大单列 j 的计算 (G[:,j] ∈ ℝⁿ, X₁ ∈ ℝⁿ, n = 4):
  Cov(G[:,j], X₁) = (1/n) Σi (Gij − Ḡ·j)(X1,i − X̄1)   (标量)
  Var(G[:,j]) = (1/n) Σi (Gij − Ḡ·j)²   (标量)
  β̂x,j = Cov / Var   (标量)

矩阵视角: 把 G 的列 j 拿出来 (n × 1), 跟 X₁ (n × 1) 做内积 — 都是同一个样本维度 n 上的 reduction.
G (列 j 被 highlight) X₁ / Y / β̂ 向量 外层 loop: j = 1..p (一次 column reduce) 内层: Cov/Var 在样本维度 n 上 reduce
看具体数字 (示范 SNP c1 和 r1) →

对每个 SNP j: $\hat\beta_{x,j} = \dfrac{\mathrm{Cov}(G_{:,j}, X_1)}{\mathrm{Var}(G_{:,j})}$    (population formula, 分母用 n=4)

SNP c1 (公共, MAF=0.30): G[:,c1] = [1, 0, 2, 1]

mean(G_c1): (1+0+2+1)/4 = 1.0
centered G_c1: [0, −1, 1, 0]
mean(X₁): (0.21 + (−0.15) + 0.19 + (−0.02))/4 = 0.23/4 = 0.0575
centered X₁: [0.1525, −0.2075, 0.1325, −0.0775]
Cov: [0·0.1525 + (−1)·(−0.2075) + 1·0.1325 + 0·(−0.0775)] / 4
    = (0 + 0.2075 + 0.1325 + 0) / 4 = 0.340/4 = 0.085

图: RARE 的因果结构 — 所有通道一张图

把 Step 1–13 用到的所有变量画成因果图. 关键问题: 从 G 到 Y 一共有 五 条路径, 其中只有 绿色 那条 (G → X₁ → Y, 系数 δ·β₁) 是想识别的真因果效应; 其余四条都是 nuisance.

5 条 G → Y 通道, 一句话总结

通道数学路径RARE 想干嘛
① TARGET G → X₁ → Y  (系数 δ · β₁) 识别 β₁ — 唯一想要的
② TARGET G → X₂ → Y  (系数 γ₂ · β₂) β₂ 已知固定, 帮助识别 β₁
③ CHP G → X₁ via κₓ  +   G → Y via κᵧ   (相关 ρ_κ) 显式建模; 通过 spike-and-slab 把两端的 κ 一起估出来
④ UHP G → Y via θ   (X 端没有对应) 显式建模; 跟 κ 独立, 用单独的 prior
⑤ 混杂 U → X₁ via ψₓ  +  U → Y via ψᵧ 潜变量; Gibbs 一起采 ψ_x, ψ_y, U
为什么这张图比 6 条结构方程更有用?

结构方程 (Step 10、11、12) 把每一项的代数写得很全, 但把 关系 藏在符号里. 这张图把同样的信息重排到 "节点–通道" 视角:

  • CHP vs UHP 的差异变得肉眼可见: CHP 是一条 "进 G 出两端" 的相关通道 (κₓ 和 κᵧ 之间有 ρ_κ); UHP 只在 Y 端有一条孤立的直接通道 (θ).
  • X₁ 和 X₂ 的不对称性显现: X₂ 这一臂只有 1 条进入箭头 (γ₂), 而 X₁ 有 3 条 (δ, κₓ, U). 这就是 RARE 把 X₁ 设成 "想识别的脏臂", X₂ 设成 "干净对照臂" 的设计.
  • U 的可怕之处: 一旦 U 同时影响 X₁ 和 Y, X₁→Y 的边缘关联里就混入 "U 制造的协方差", 这就是经典的 backdoor confounding. IV 方法 (含 RARE) 全靠 G ⊥ U 才能切开这条 backdoor.
  • 识别 β₁ 的 IV 假设的图形表达: relevance = G→X₁ 那条 δ 箭头存在; exclusion = G 没有直接到 Y 的箭头 (← 但 κ 和 θ 让这条假设破了, 所以 RARE 必须显式建它们); exchangeability = G ⊥ U (G 和 U 之间没有箭头).

Section 2 · MR-CARV 玩具版

MR-CARV 的仿真 故意简化: 没有混杂 U, 没有 CHP, 没有 UHP. 它的核心创新点是 burden 怎么构 + 每个 rare SNP 是否 causal 用 Bernoulli 抽 + annotation 进 logit. 所以下面 X 和 Y 都比 RARE 短得多.

Step 6

组装 X — 矩阵形状视角 (含 Hadamard 罩 φ)

$$ \underbrace{X}_{n \times 1} \;=\; \underbrace{G_c\,\alpha_c}_{(n \times p_c)(p_c \times 1)} \;+\; \underbrace{G_r\,(\varphi \odot \alpha_r)}_{(n \times p_r)(p_r \times 1)} \;+\; \underbrace{\varepsilon}_{n \times 1} $$

两个 matrix-vector 积 + 噪声; 关键: rare 那一支的系数是 Hadamard (element-wise) 乘积 $\varphi \odot \alpha_r$ — Bernoulli 罩 $\varphi$ 把 $\alpha_r$ 里 "不 causal" 的位置整列归零.

Hadamard 含义 (一维向量, 长度 p_r = 4):
  (φ ⊙ αr)j = φj · αr,j   (j 一一对应, 没有 sum)
  φj = 0 ⇒ 第 j 个 rare SNP 整条系数被关掉.

然后单样本 i:
  Xi = (Gc αc)i + (Gr(φ⊙αr))i + εi
  = Σj∈c Gcij αcj + Σj∈r Grij (φj αrj) + εi
Gc, Gr αc, αr, φ ε X · matrix-vector ⊙ Hadamard (element-wise mul) + element-wise add
看具体数字 (seed=1, n=4) →

$$ X_i \;=\; \sum_{j \in \text{common}} \alpha_{c,j} G_{ij} \;+\; \sum_{j \in \text{rare}} (\varphi_j \alpha_j) G_{ij} \;+\; \varepsilon_i $$

Step 8

Burden 构建 — 矩阵形状视角 (rare 列被一个 weight 向量 collapse 成单列 B)

$$ \underbrace{B}_{n \times 1} \;=\; \underbrace{G_r\, w_r}_{(n \times p_r)(p_r \times 1)} $$

关键操作: 把 (n × p_r) 的 rare 基因型矩阵, 通过 right-multiply 一个长度 p_r 的 weight column vector, collapse 成长度 n 的 burden score 向量. 这就是 "burden test" 名字的来源 — 把多个 rare variant 压成 1 个 summary score.

放大单样本 i: Bi = Σj=1..4 Grij · wj
  = 一行 (n × 1 切片的第 i 行) 跟 weight 向量做内积 → 1 个标量
Gr (rare 基因型) w (Beta 权重向量) B (burden score) · matrix-vector (collapse p_r → 1)
Beta(1,25) 怎么选的?
这是 STAAR 默认权重. Beta(1,25) 的 pdf 在 MAF=0 附近最大, MAF=0.1 已经几乎 0, 所以"越罕的等位携带更大权重". MR-CARV 还会同时跑 Beta(1,1) (等权重), 最后用 Cauchy combination 合并 p-values.
看具体数字 (seed=1, n=4) →

权重 $w_j = \mathrm{Beta\text{-}pdf}(\mathrm{MAF}_j; 1, 25) = 25 \cdot (1-\mathrm{MAF}_j)^{24}$ — 越罕权重越大.

w₁: 25 · (0.995)²⁴ = 25 · 0.8866 = 22.16
w₂: 25 · (0.992)²⁴ = 25 · 0.8246 = 20.62
w₃: 25 · (0.990)²⁴ = 25 · 0.7857 = 19.64
w₄: 25 · (0.997)²⁴ = 25 · 0.9304 = 23.26

$$ B_i \;=\; \sum_{j=1}^{4} w_j \, G^{\text{rare}}_{ij} $$

s2: 22.16·1 + 20.62·0 + 19.64·0 + 23.26·0 = 22.16

Step 10

IVW — 矩阵形状视角 (把 IV 维度 K = p_c + 1 collapse 成标量)

$$ \hat\beta_{\mathrm{IVW}} \;=\; \frac{\hat\beta_x^{\top}\, \mathrm{diag}(w)\, \hat\beta_y}{\hat\beta_x^{\top}\, \mathrm{diag}(w)\, \hat\beta_x} $$

Stack 所有 IV 的 summary stats 成两个向量: $\hat\beta_x = [\hat\beta_x^{c_1}, \hat\beta_x^{c_2}, \hat\beta_x^{B}]^{\top} \in \mathbb{R}^{K}$ 和 $\hat\beta_y \in \mathbb{R}^{K}$ (这里 K = p_c + 1 = 3 = 2 common + 1 burden). 然后两个加权内积 (= 标量) 相除.

Stack 顺序 (玩具版): β̂ₓ = [β̂_x,c1, β̂_x,c2, β̂_x,B]⊤, β̂ᵧ = [β̂_y,c1, β̂_y,c2, β̂_y,B]⊤
Burden 跟 common 在同一个向量里 — 一个 IV 占一个 slot, 不区分类型.

展开内积 (K = 3):   β̂ₓᵀ diag(w) β̂ᵧ = Σk=1..K wk β̂x,k β̂y,k
β̂ₓ, β̂ᵧ (IV 系数向量) diag(w) (权重矩阵) β̂_IVW (输出标量) · matrix-vector diag(·) = 向量上对角化成方阵

(真值 β = 0.04. 玩具版 n=4 信噪比太低, 估值离真值远是预期; 真实 n=50000 才有 power.)

(主仿真有 4 common + 4 burden = 8 IV; 玩具只有 3, 但 IVW 公式不变.)

$$ \hat\beta_{\mathrm{IVW}} \;=\; \frac{\sum_k w_k \,\hat\beta_{x,k}\,\hat\beta_{y,k}}{\sum_k w_k\,\hat\beta_{x,k}^{\,2}} $$

玩具没有 SE, 用 unit weights $w_k = 1$:

分子: 0.299·0.047 + (−0.299)·(−0.022) + 0.0189·(−0.000351)
    = 0.01405 + 0.00658 + (−0.0000066)

Step 11

跟 RARE 的对比

看完 RARE 的 14 步和 MR-CARV 的 10 步, 该看清的:

  • RARE 显式建 3 层干扰: U (混杂) → ψ → X 和 Y; G → κ → X 和 Y (CHP); G → θ → Y (UHP).
  • MR-CARV 啥都没建: Y = β X + ε, X = Σα G + ε. 没有 U, 没有 G→Y 直接路径.
  • 所以同一份 n=50000 仿真里, RARE 才能测出 "在 pleiotropy 存在时是否还能识别 β"; MR-CARV 测不了 — 因为它的仿真没生这个场景.
关键对比

Section 3 · 一张表看完两者的差别

变量 inventory

只在 RARE 里有

  • U — 潜在混杂
  • κₓ, κᵧ — CHP 系数
  • θ — UHP 系数
  • ψₓ, ψᵧ — 混杂载荷
  • β₂, X₂ — 次曝光通道
  • γ₂ — common-IV 通道

两个都有 (共享骨架)

  • G — 基因型矩阵
  • X — 暴露
  • Y — outcome
  • ε — 高斯噪声
  • α 或 δ — IV → X 系数
  • summary stats (β̂_x, β̂_y)

只在 MR-CARV 里有

  • φ — Bernoulli causal indicator
  • annotation 矩阵 A
  • b₀, b_i — logit 系数
  • burden 权重 w_j = Beta(MAF; 1,25)
  • burden score B
  • α = c₀·|log₁₀ MAF| 缩放
一句话: RARE 在仿真里建了 pleiotropy 通道 (G → Y 直接, 通过 κᵧ 和 θ) 以及混杂通道 (U → X, U → Y 共同祖先), 所以它的仿真会把 "valid-instrument 假设破坏掉" 这个最常见的失败模式生进数据; MR-CARV 完全没建 — Y = β X + ε — 这是为什么它的 Type-I 看着好, 因为它根本没测 pleiotropy 场景. 所以两者的"Type-I 通过"含义完全不一样, 不能直接比较.
