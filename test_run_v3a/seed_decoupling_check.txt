Seed-decoupling verification (CRITIQUE_v2 §S2.v2.1 fix)
========================================================

v2 per-rep seed formula (run_tests_v2.R:150):
  seed_r = ((master_seed + scenario_idx_offset) %% 1e6) * 1e4 + r

This creates exact collisions whenever
  (master_seed_A + idx_A) == (master_seed_B + idx_B)
i.e. any two scenarios at adjacent idx and adjacent seeds.

=== v2 documented collision example ===

Fsweep_F20 (scenario index 6) at master_seed 20260604, rep 1:
  v2 seed = ((20260604 + 6) %% 1e6) * 1e4 + 1 = 458616354

Plei_mult0 (scenario index 7) at master_seed 20260603, rep 1:
  v2 seed = ((20260603 + 7) %% 1e6) * 1e4 + 1 = 458616354

THESE ARE THE SAME SEED.  This matches the v2 anchors_run.log
observation noted in CRITIQUE_v2 §S2.v2.1: Plei_mult0 seed-1
(F=20, cs=0, no plei) produced cov_AR=0.940, F_mean=21.24,
n_bd=296, BIT-IDENTICAL to Fsweep_F20 seed-2.

=== v3a hash-based per-rep seed (run_tests_v3a.R:hash_seed) ===

  seed_r = digest::digest2int(paste(scenario_name, master_seed, r, sep="|"))
           %% .Machine$integer.max  (mapped to positive)

For the same scenario+seed pair as above:
  Fsweep_F20 + master_seed 20260604 + rep 1 -> seed 175911825
  Plei_mult0 + master_seed 20260603 + rep 1 -> seed 1491561288
  NOT IDENTICAL.

The hash mixes scenario name into the seed, breaking the modular-linear
collision pattern of v2.
